# GitHubCN 后续词条补充建议

以下页面结构更新频繁，建议后续持续补充：

- 新版全局搜索（/search）高级筛选区（Code Search Beta）
- Projects v2 视图配置弹窗（字段、分组选项、自动化规则）
- Actions 运行日志中的侧边栏状态文案
- Discussions 富文本编辑器工具栏提示
- Organization 设置页中的团队与安全策略区块
- 仓库 Settings 下 Security / Code scanning 细分页面

## 建议补充策略

1. 优先补充导航、按钮、筛选器、空状态文案。
2. 新增词条时尽量使用完整短语，减少单词级词条导致的误翻译。
3. 每次 GitHub 大版本 UI 更新后，至少抽样检查以下页面：
   - 仓库首页
   - Issues 列表与详情
   - Pull Requests 列表与详情
   - Actions 列表与运行详情
   - Discussions 列表与详情
